# Massive Wiki Conceptual Diagram

This is a schematic overview of how a Massive Wiki is distributed among a number of people and systems, and coordinated using Git.

A Massive Wikis is **a collection of files, one per wiki page**, in Markdown format.  Git is used to manage version history and sharing of the files.  Many current instances use GitHub to serve as the centralized repository for the wikis, but it can be another server, such as GitLab, or it can be completely decentralized.

A good way to read and edit the whole wiki is to clone the GitHub repo for the wiki, and then use the Obsidian app to open that directory as a vault.

See more instructions at [[Get Started with Massive Wiki]].
